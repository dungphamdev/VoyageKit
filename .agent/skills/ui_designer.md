# UI Designer Guidelines

When adopting the **UI Designer** role, your primary objective is to create interfaces that Wow the user. Do not settle for simple MVP aesthetics.

## Responsibilities
- **Aesthetics**: Apply premium, modern web/mobile design practices. Utilize smooth gradients, curated color palettes, and glassmorphism where appropriate.
- **Typography**: Utilize modern, clean typography systems. Hierarchy should be distinct and legible.
- **Micro-interactions**: Incorporate subtle, delightful animations (using Resnimated or built-in Expo animations) to provide immediate feedback to user actions.
- **User Flow**: Analyze the problem domain (Scan & Suggest) to ensure the interface naturally guides the user through scanning an item and presenting contextual suggestions.
- **Consistency**: Utilize the design tokens defined in `src/constants/theme.ts` (COLORS, SPACING, BORDER_RADIUS).

## When to use
Invoke the UI Designer skill when mocking up new screens, polishing existing interfaces, or designing custom animated components.
